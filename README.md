# xm3
项目功能介绍​
​
项目概述​
Sagt是一个基于LangGraph架构实现的零售行业销售智能体系统，旨在通过智能分析客户信息，协助员工更好地维护客户关系，提升销售效率和客户服务质量。

核心功能模块​
​
1. wechat_client - 微信API客户端服务​
核心职责：封装微信生态体系API，提供统一的数据访问接口​
​
主要功能：​
•
数据获取：获取企业微信员工信息、客户联系信息、标签体系、客户聊天记录、客服交互记录、微信小店订单数据​
•
数据操作：设定企业员工工作日程、管理客户标签分类​
•
接口封装：提供标准化的API接口，简化微信生态数据访问复杂度​
​
2. wechat_notify - 微信事件监听服务​
核心职责：构建事件驱动的数据同步机制，实现微信生态数据的实时感知​
​
主要功能：​
•
事件监听：实时监听外部联系人信息变更、企业标签体系变更、企微客户会话事件、微信客服事件、微信小店订单事件​
•
事件分发：将监听到的事件及时分发给相关业务模块​
​
3. wechat_sync - 数据同步服务​
核心职责：建立微信生态与Sagt智能体系统之间的数据桥梁​
​
主要功能：​
•
多模式同步：支持事件通知驱动的实时同步和定时补偿的批量同步​
•
数据转换：将微信生态数据格式转换为智能体系统可识别的格式​
•
同步调度：通过调用wechat_client获取数据，经由sagt_client写入智能体系统​
•
同步监控：监控数据同步状态，确保数据一致性​
​
4. sagt_client - 智能体API客户端​
核心职责：封装Sagt智能体系统API，提供标准化的数据访问和操作接口​
​
主要功能：​
•
API封装：提供智能体系统的统一访问接口​
•
数据管理：支持智能体数据的读取、写入和更新操作​
•
记忆维护：在智能体系统中维护客户关系的长期记忆​
​
5. sagt_admin - 智能体管理后台​
核心职责：提供Sagt智能体系统的可视化管理和监控界面​
​
主要功能：​
•
员工管理：查看和管理授权员工信息及权限配置​
•
客户管理：查看员工客户信息、客户画像和关系状态​
•
会话监控：查看员工与客户的会话记录和沟通历史​
•
服务记录：查看客户客服交互记录和服务质量分析​
•
订单管理：查看客户微信小店订单信息和购买行为​
​
6. sagt_agent - 核心智能分析引擎​
核心职责：基于客户数据分析，为员工提供智能化的客户关系维护支持​
​
主要功能：​
•
智能聊天建议：基于客户历史信息和行为模式，生成个性化的沟通建议和话术推荐​
•
自动标签标注：运用AI算法分析客户特征，自动为客户标注合适的标签分类​
•
智能日程管理：根据客户交流内容和重要程度，自动为员工安排工作日程和提醒​
•
客户画像提取：深度分析客户数据，自动生成详细的客户Profile，包括偏好、需求、价值等信息​
​
7. sagt_sidebar​
核心职责：提供客户交互页面，审核Agent操作（人工介入管控）​
​
主要功能：​
•
智能对话界面： 提供直观的Web界面，支持员工与SAGT智能体进行自然语言交互​
•
菜单化操作： 预设常用业务场景菜单，包括客户画像分析、标签建议、聊天建议、客服建议、日程建议等​
•
实时流式响应： 支持智能体响应的实时流式展示，提供良好的用户体验​
•
中断确认机制： 当智能体需要人工决策时，提供中断确认界面，支持确认、放弃、重新生成等操作​
​
技术架构特点​
​
1.
事件驱动架构：通过事件监听实现实时数据同步​
2.
模块化设计：各功能模块独立，便于维护和扩展​
3.
智能体记忆：基于LangGraph实现长期记忆管理​
4.
API封装：统一的API接口，便于系统集成​
5.
数据完整性：多重同步机制确保数据一致性​
​
这个架构设计充分体现了现代智能体系统的特点，既保证了数据的实时性和完整性，又提供了强大的智能分析能力，为零售行业的销售工作提供了有力的技术支撑。
